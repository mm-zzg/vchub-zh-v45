# 管理平台

为了让组态人员能够更专注于组态工作，WAGO VC Hub将其他组态相关的前置工作都归类放入了我们的管理平台。在开始您的正式组态工作之前，您首先需要在管理平台对即将组态的项目进行一些配置。

以下是WAGO VC Hub管理平台的一些重要功能，这些功能的具体介绍可以点击相应链接查看。

| **功能** | **描述**                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|----------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 项目     | WAGO VC Hub支持您同时创建多个组态[known-link] ，每个项目都可以包含各自独立的画面，但是所有项目都共享管理平台的所有配置，比如可以共享变量、设备连接、数据库连接和网络配置等。                                                                                                                                                                                                                                                                                     |
| 设备     | 设备是指在您组态项目时，您需要连接的现场设备[known-link] ，WAGO VC Hub可以通过多种现场总线协议连接您的各种设备。比如您现场有大量的Modbus TCP设备，那么您就可以使用WAGO VC Hub设备中提供的Modbus TCP连接来创建对这些现场的设备连接。                                                                                                                                                                                                                             |
| 变量     | 不同的设备（或者说是不同的现场总线协议），有不一样的地址空间，为了处理这种不同，WAGO VC Hub提供了[known-link] 的概念和功能，变量是WAGO VC Hub平台相关的，而不是设备（协议）相关的，但是一个变量可以通过绑定跟各种设备的地址进行绑定。比如一个温度变量，其可以绑定到一个Modbus设备中的地址，也可以绑定到OPC UA设备中的一个地址。更重要的是，WAGO VC Hub支持资产化构建您的变量系统，以便其可以和您的实际现场设备物理模型完美匹配，方便您后续的管理和操作。                     |
| 报警     | 在VC Hub系统中，[known-link] 是非常重要也是必不可缺少的功能。WAGO VC Hub像您提供了基于行业标准的报警功能，但在您组态报警前，您需要先在管理平台进行一些报警相关的前置配置工作。比如，报警需要最终进行存储，所以为了存储这些报警，您需要先配置报警对应的[known-link] 。另外对于陈旧报警的时长，您也可以进行配置。                                                                                                                                                       |
| 脚本     | [known-link] 是VC Hub系统中的高级功能，在一些复杂业务场景，脚本能让您轻松实现组态无法实现的功能，比如您需要一些跟变量相关的逻辑算法。WAGO VC Hub使用简单高效的Javascript作为脚本语言，并且提供了多场景下的脚本支持。您在管理平台配置的脚本都在服务器端运行（WAGO VC Hub是以Web Server方式在服务器端运行的），比如变量变化脚本、周期性脚本以及时间表（用时间表的形式来运行脚本）。当然我们也在您组态时，提供跟您页面组态息息相关的脚本功能，比如控件脚本、动作脚本等。       |
| 网络     | 网络配置为您提供了[known-link] ，[known-link] 等配置功能。在很多场景，您的VC Hub需要可靠稳定的运行，那么主备冗余是必不可少的功能，您能通过WAGO VC Hub提供的简单配置，快速构建您的高可用场景。另外，如果您有大规模的应用场景，比如多条生产线，多个分厂分公司等需要监控，那么WAGO VC Hub的[known-link] 功能就为您提供了这样的能力，您可以将多套WAGO VC Hub分别部署在不同的产线或者分公司，然后再将他们组合在一起使用。当然WAGO VC Hub也提供了网络传输之间的安全配置等功能。 |
| 数据库   | WAGO VC Hub支持您使用多种数据作为历史数据或者报警数据的存储库 注1。除了传统的关系型数据库，如MySQL、SQL Server等，也支持更高性能的时序数据库InfluxDB。对于历史数据存储，WAGO VC Hub也支持多种模式，比如单历史库存储，双历史库存储，或者远端历史库存储。                                                                                                                                                                                                              |
| 权限     | WAGO VC Hub提供了基于[known-link] 的角色管理系统，您可以为您的现场应用构建各种角色，比如工程师、操作员、维护员等。为了便于您使用，WAGO VC Hub也给您内置了几个常用的角色供您直接使用。                                                                                                                                                                                                                                                                             |
| 工程     | 为了方便组态人员操作，WAGO VC Hub允许您构建多个[known-link]   注2。但是同一时刻，组态人员只能操作一个工程。如果您需要操作其他工程，必须在这些工程间进行手工切换 注3。                                                                                                                                                                                                                                                                                            |
| 诊断     | VC Hub是一套复杂的系统，涉及到各个方面，所以为了监测和诊断WAGO VC Hub系统本身，其提供了丰富的[known-link] 功能，比如您可通过诊断实时查看WAGO VC Hub系统占用的CPU、内存等情况，也可以实时查看WAGO VC Hub的运行性能等。                                                                                                                                                                                                                                             |

注1：WAGO VC Hub只内置了一个SQLite数据库，用于存储系统必要的配置信息等，所以WAGO VC Hub本身不提供任何数据库，而只提供对这些数据库的连接和访问能力，所以您需要自行配置诸如MySQL、SQL Server等这些第三方数据库。

注2：工程是项目的集合，一个工程中可以包含多个项目，这些项目共享同一份系统配置，比如设备配置、变量配置，数据库配置等。

注3：手工切换工程后，很可能需要您重新登录系统，因为不同工程之间用户是独立的，即便您在A工程中有个用户UserA，在B工程也存在一个同名的UserA，这2个用户之间也不存在任何关联。
